Diagram Automation Engine

Tell the story of your architecture with Nagare.

Nagare is Saasuke Labs' declarative diagramming engine for event-driven architectures, distributed systems, and runtime workflows. Transform a concise DSL into polished SVG diagrams that stay in sync with the systems they explain.

Why teams choose Nagare

Purpose-built for architecture

Capture services, queues, functions, and lifelines with the same vocabulary your engineers use. Say goodbye to dragging shapes around generic design tools.

Readable, expressive DSL

Describe flows using a Markdown-like syntax that is optimized for humans and machines, enabling code reviews and documentation diffs.

Consistent visuals

Every render uses a shared component library and layout rules, producing diagrams that look great and stay consistent across teams and repos.

Automation ready

Integrate rendering into CI, docs pipelines, and design reviews through the lightweight Go service or Docker image.

How Nagare works

  1. Write the flow in a .mmd-style document describing components, states, and relationships.
  2. Send it to Nagare via the HTTP /render endpoint or Go API.
  3. Receive polished SVG diagrams ready to embed in READMEs, docs portals, and design reviews.
@state(client)
  Browser "User"
  -> API "Gateway"

@state(processing)
  Function "Order Processor"
  -> Queue "Orders"
  Queue -> Service "Inventory"

Nagare parses the DSL, builds a component layout, and renders an SVG that mirrors the logical flow of your architecture.

Sample rendering pipeline

POST /render
{
  "dsl": "@state(client)..."
}

The /render endpoint accepts DSL input and returns SVG output. Embed the SVG in Saasuke Labs, Confluence, or any static site generator.

Common use cases

  • Onboarding diagrams for new services and platform capabilities.
  • Visualizing message-driven microservice interactions.
  • Architecture decision records and ADR appendices.
  • Runbooks for operational procedures and failover paths.

Get started

Explore sample flows

Review example.json, example2.json, and scene.json in the repository for ready-made DSL.

Run the renderer

Launch the service with go run ./cmd/main.go and POST DSL content to http://localhost:8080/render.

Embed the SVG

Drop the generated SVG into READMEs, documentation sites, or Saasuke Labs knowledge bases.

Automate updates

Wire Nagare into CI to regenerate diagrams whenever your DSL changes, keeping visuals aligned with code.
